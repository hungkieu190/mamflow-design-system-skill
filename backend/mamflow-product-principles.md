# MamFlow Product Principles

You are building products for WordPress professionals.

Our users include:

* Site Owners
* Agencies
* Freelancers
* Developers
* Course Creators
* Membership Businesses

They use our products to run businesses.

Never treat users as casual consumers.

---

# Core Beliefs

## Respect Time

Every click costs time.

Every screen should help users complete tasks faster.

Remove unnecessary steps.

---

## Clarity Over Creativity

Do not introduce visual elements simply because they look modern.

Prioritize understanding.

Users should immediately understand:

* What happened
* Why it happened
* What to do next

---

## Function Before Decoration

A feature is complete only when users can operate it efficiently.

Visual polish is secondary.

---

## Consistency Wins

If a pattern already exists in the product:

Reuse it.

Do not invent new patterns.

---

## Reduce Cognitive Load

Users should not need to remember information between screens.

Surface important information where decisions are made.

---

# Every Screen Must Answer

1. What is this?
2. What can I do here?
3. What should I do next?

If a screen cannot answer these questions, redesign it.

---

# Anti-Patterns

Avoid:

* Marketing-style dashboards
* Large hero sections
* Fancy animations
* Decorative charts
* Excessive colors
* Deep navigation trees

The product is a tool, not a landing page.
